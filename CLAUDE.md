## Git 運用ルール

**コードを変更するたびに、必ず GitHub へプッシュすること。**

具体的な手順:

1. 変更を加えたら、意味のある単位で `git add` する。
2. 日本語で分かりやすいコミットメッセージを付けて `git commit` する。
3. **コミット後はすぐに `git push` で GitHub に反映する。**

```bash
git add -A
git commit -m "変更内容を説明するメッセージ"
git push
```

補足:

- 1回の作業（タスク）が完了したら、ローカルに変更を溜め込まず必ずプッシュして同期状態を保つ。
- まだ Git リポジトリが初期化されていない場合は、最初に以下を行う:

  ```bash
  git init
  git remote add origin <GitHubリポジトリのURL>
  ```

- コミットメッセージは「何を・なぜ」変更したかが分かる粒度にする。
- `main` ブランチへ直接プッシュするか、ブランチを切って作業するかは、運用が固まった時点で本ルールを更新する。
