# 事業検証レポート：MeetMemo

**サービス概要：** 会議録音をAIが自動で議事録化し、抽出したタスクを担当者へ自動通知するSaaS
**作成日：** 2026年6月23日
**ステータス：** 初期事業検証（Web調査ベース）

---

## エグゼクティブサマリー

- **課題仮説は概ね妥当。** 「ムダな会議」「議事録作成の負荷」「会議後のタスク抜け漏れ」は、複数の調査で定量的に裏付けられた普遍的な課題である。
- **市場は高成長。** AI議事録・会議アシスタント市場は世界で年率約25%成長、2024年の約28億ドルから2034年には約270億ドル規模へ。国内の音声認識市場も2023年度150億円から2028年度300億円超へと拡大予測。
- **競合は強力だが、明確な空白地帯がある。** Otter.ai・Notta・Fireflies.ai はいずれも「文字起こし＋要約」までは高水準。一方で **「タスクを担当者に確実に届け、実行まで追跡する」フォローアップ領域** は各社とも作り込みが浅く、ここが MeetMemo の勝ち筋となり得る。
- **差別化の核：** ①タスク実行管理への深掘り、②日本語・国内業務ツール連携の最適化、③「議事録作成」ではなく「会議後の実行」へKPIを置いたプロダクト設計。

---

## 1. 課題仮説の検証

### 仮説
「会議の議事録作成は負荷が高く、会議で決まったタスクは担当者に正しく伝わらず・実行されないことが多い」

### 検証結果：**支持される（確度：高）**

#### (1) 会議そのものがムダだと認識されている
- パーソル総合研究所の調査では、**1万人規模の企業でムダな社内会議は年間約67万時間（約332人分の年間労働時間に相当）、損失額は年間約15億円**に及ぶと試算されている。
- 会議をムダだと感じている割合は、メンバー層で23.3%、上司層で平均27.5%。
- 社内会議中にムダだと感じる点の上位は、1位「長時間の会議（54.3%）」、2位「議題が不明瞭（44.5%）」、3位「不要な人員が多い（33.6%）」。
- 特にムダ指数に強く影響するのは **「会議が終わっても何も決まっていない」「終了時刻が延びる」** といった“終わり方”に関わる要素であり、これは「決定事項・タスクの曖昧さ」という MeetMemo が狙う課題と直結する。

#### (2) 議事録作成の工数負荷が大きい
- 導入企業の声として「会議時間の2〜3倍かかっていた議事録作成が大幅に短縮できた」という実績が報告されており、作成工数が会議時間を上回るケースが存在する。
- 議事録は「決定事項や担当者・期限付きのアクションを、誤記や漏れがないか念入りに確認する」必要があり、手作業では漏れが生じやすいことが業務解説でも指摘されている。

#### (3) 会議後のタスク抜け漏れ・フォローアップが弱い
- 業務ノウハウの解説でも「タスクの担当者・期限を明確にし、スケジュール管理ツールと連携させることで抜け漏れを防ぐ」ことの重要性が繰り返し説かれており、裏を返せば **現状は抜け漏れが頻発している** ことを示す。
- 「決定事項やToDoをすぐ共有・実行に移せること」がAI議事録ツールの主要な期待価値とされている。

#### (4) AIへの受容性が高い
- 企業の78%がAIを業務に活用、71%が生成AIを定期利用、96%のマネージャーが部下のAI利用を認識し75%が推奨——導入の心理的障壁は急速に下がっている。

### 示唆
課題は実在し、かつ「議事録作成」より一段深い **「会議後の意思決定・タスク実行の不全」** こそが損失の本丸である。MeetMemo の「タスク自動通知」コンセプトは、この本丸に踏み込んでいる点で筋が良い。

## 2. 市場規模の推定

### 市場トレンド
- **AI会議アシスタント／AI議事録市場（世界）：** 2024年 約27.8億ドル → 2034年 約272.9億ドル、**CAGR 約25.6%**（複数調査が24〜26%でほぼ一致）。
- **国内音声認識市場：** 2023年度 売上150億円（前年比+21.0%）→ 2024年度 +18.0%、**2028年度には300億円超**の予測（ITR）。
- **国内生成AI市場：** 2024年に初めて1,000億円超（1,016億円）に到達見込み。
- リモート／ハイブリッドワークの定着と自然言語処理の高度化が主成長ドライバー。北米が世界売上の約35%を占める。

### TAM / SAM / SOM 概算（試算・前提付き）

| 区分 | 定義 | 概算規模 | 算出の考え方 |
|---|---|---|---|
| **TAM** | 世界のAI会議アシスタント／議事録市場全体 | **約28億ドル（2024）→ 270億ドル（2034）** | 上記調査の市場規模をそのまま採用 |
| **SAM** | 国内＋日本語ビジネス会議の議事録・タスク管理領域 | **約300〜500億円規模（2028目安）** | 国内音声認識市場300億円超に、隣接するタスク管理・通知連携の付加価値分を加味した推定レンジ |
| **SOM** | MeetMemo が3年で現実的に獲得しうるシェア | **約3〜15億円／年** | SAMの1〜3%を初期獲得目標と仮定（新興SaaSの現実的レンジ） |

> ※TAM以外は本調査時点での前提付き試算であり、ターゲットセグメント（中小企業／エンタープライズ／特定業種）の確定後に再計算が必要。

### 国内競争環境（シェア参考）
議事録ツールの国内シェア調査（1,690人）では、Otolio（旧スマート書記）18.8%、Notta 12.1%、AI GIJIROKU 11.3%、YOMEL 9.3%、Rimo Voice 6.8% と続き、**寡占化しておらず参入余地がある**。

### 示唆
市場は高成長かつ未寡占。ただし「文字起こし・要約」は早晩コモディティ化するため、**価格競争を避けられる上位レイヤー（タスク実行・ナレッジ活用）**での価値訴求が中長期の生存条件となる。

## 3. 主要競合3社の調査

### ① Otter.ai（米国・グローバル最大手の一角）
| 項目 | 内容 |
|---|---|
| 概要 | 英語圏で高シェアのAI会議アシスタント。リアルタイム文字起こし、OtterPilotによる自動参加、CRM連携（HubSpot等） |
| 料金 | Free（300分/月）／Pro $8.33〜16.99/月（6,000分）／Business $20〜30/席/月／Enterprise 個別見積（SSO・コンプライアンス） |
| 強み | ブランド力・英語精度・Zoom/Meet/Teams連携、営業向けOtterPilotなどユースケース特化 |
| 弱み | **日本語の精度・UIが弱い**、タスク管理は要約内の項目抽出にとどまる、国内業務ツール（Slack日本利用やChatworkなど）連携の最適化が薄い |

### ② Notta（日本市場で強い新興）
| 項目 | 内容 |
|---|---|
| 概要 | 多言語対応のAI文字起こし／議事録。国内シェア上位（約12%）。専用ハード・多言語でグローバル強化中（9.9億円調達） |
| 料金 | Free（120分/月）／有料 1,100〜3,300円/月（24〜120時間）。Free/Pro/Business構成 |
| 強み | **日本語精度と国内認知**、カレンダー連携、Zapier経由で1,000以上のツール連携、「決定事項／要確認／タスク」のタグ付け |
| 弱み | タスクは**タグ付け止まりで「担当者への自動アサイン・通知・実行追跡」までは弱い**。連携も汎用（Zapier依存）で、専用の実行管理機能は限定的 |

### ③ Fireflies.ai（米国・タスク機能が比較的強い）
| 項目 | 内容 |
|---|---|
| 概要 | AI会議ノート＋自動アクションアイテム抽出。Slack/Notion/Salesforce/HubSpot/Trello/Asana等にネイティブ連携 |
| 料金 | Free（800分保存）／Pro $10〜18/月／Business $19〜29/月／Enterprise $39〜（HIPAA対応）。AI機能は別途クレジット課金 |
| 強み | **アクションアイテムの自動抽出＋担当者アサイン（言及時）**、リアルタイムノート、豊富なネイティブ連携 |
| 弱み | **日本語対応・国内サポートが弱い**、AI機能がクレジット従量課金で**コストが読みにくい**、抽出はするが「実行されたか」の追跡・リマインドは弱い |

### 競合まとめ
- 「文字起こし＋要約」は3社とも高水準で**差別化が困難**。
- Fireflies がタスク抽出で一歩進むが、**日本語・国内連携が手薄**。
- Notta は日本語で強いが、**タスク実行管理が浅い**。
- → **「日本語×タスク実行管理の深さ」を両立するプレイヤーが不在**。

## 4. 差別化ポイントの整理

競合比較から、MeetMemo が勝てる軸を3つに絞る。

### 軸① タスクの「実行」まで責任を持つ（議事録作成ツール → 実行管理プラットフォーム）
- 競合は「タスクを抽出・表示」して終わり。MeetMemo は **担当者への自動通知 → 期限管理 → 未完了リマインド → 完了報告の回収** までを一気通貫で設計。
- KPIを「議事録作成時間の削減」ではなく **「会議で決まったタスクの実行完了率」** に置く。これは課題検証(1)で示された“何も決まらない・実行されない会議”という損失の本丸を直撃する唯一の軸。

### 軸② 日本語・国内業務スタックへの最適化
- 日本語の発話（敬語・主語省略・複数話者の曖昧な担当者指定）から **「誰が・何を・いつまでに」を高精度に構造化**。
- 通知先を **Slack / Microsoft Teams / Chatwork / LINE WORKS / メール** など国内で実際に使われるチャネルにネイティブ対応（Otter/Firefliesの弱点、Notta も汎用Zapier依存）。

### 軸③ 透明・予測可能な価格と、タスク量に応じた価値設計
- Fireflies の「AIクレジット従量課金」は法人にとってコストが読めない弱点。MeetMemo は **席数ベースの明朗・定額** とし、稟議を通しやすくする。
- 「文字起こし分数」ではなく **「処理したタスク・通知数」=成果に連動する価値訴求** で、文字起こしコモディティ化の価格競争から脱却。

### 差別化ポジショニング（一言）
> **「議事録を作るツール」ではなく、「会議の決定事項を確実に実行させるツール」** ——日本語と国内ツール連携に最適化した、唯一のタスク実行特化型AI議事録SaaS。

---

## 補足：今後の検証推奨アクション
1. **想定顧客10〜20社へのデプスインタビュー** — 「議事録作成」と「タスク未実行」のどちらの痛みが強く、いくら払うかを定性検証。
2. **SAM/SOMの精緻化** — ターゲット業種・企業規模を絞り、ボトムアップ（社数×単価）で再算出。
3. **タスク抽出精度のPoC** — 日本語会議音声で「担当者・期限の抽出精度」を競合と比較ベンチマーク。
