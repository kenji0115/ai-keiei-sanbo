# 戦略設計レポート：MeetMemo

**事業テーマ／業界：** 議事録SaaS・業務効率化
**提供価値：** 会議の録音をAIが自動で議事録化し、タスクを抽出して担当者へ自動通知する
**ターゲット顧客：** 従業員50名以下の中小企業
**作成日：** 2026年6月23日

> 本レポートは PEST分析 → SWOT分析 → 3C分析 の順で実施。SWOTの「機会・脅威」にはPEST分析の結果を反映している。

---

## 1. PEST分析

### Political（政治・規制）
- **AI導入を国が後押し。** 2025年までの「IT導入補助金」が2026年に **「デジタル化・AI導入補助金」へ名称変更**され、AI活用が国の方針として明確化。AIを活用した業務効率化ツールが補助対象に追加され、クラウドサービス利用料も補助対象。小規模事業者は賃上げ要件等を満たすと**補助率が最大4/5**まで上昇 → ターゲットの中小企業にとって導入コストの障壁が下がる **追い風**。
- **AI規制はソフトロー中心。** 2025年6月公布・9月全面施行の「AI推進法」は**罰則のない推進法**で、ガイドラインベース。総務省・経産省の「AI事業者ガイドライン」は2026年3月に第1.2版へ更新 → 重い規制負担は当面少ない。
- **個人情報保護法の改正動向に注意。** 2025年成立・2027年施行見込みの改正個人情報保護法で生成AIの取り扱いが検討対象。**会議録音＝個人情報・要配慮情報を含みうる**ため、プライバシーポリシー策定・録音同意・データ管理体制が事業上の必須要件。
- **補助金申請には登録IT導入支援事業者とのパートナー必須** → 自社が支援事業者登録すれば販路・信頼性の武器になる。

### Economic（経済）
- **深刻な人手不足が構造化。** 2024年に**人手不足倒産342件（2年連続で過去最多）**、2025年1月時点で正社員「不足」企業が**53.4%**でコロナ後最高 → 少人数で成果を出す「省人化・効率化」ニーズが強い。
- **中小企業の生産性が伸び悩み。** 2024年度の時間当たり名目労働生産性は5,543円と過去最高だが、上昇は主に大企業。中小は伸び悩み → **中小向けの生産性向上ソリューションに政策的・経済的な需要**。
- **市場は高成長。** AI会議アシスタント／議事録市場は世界で2024年 約28億ドル → 2034年 約270億ドル（CAGR約25%）。国内音声認識市場は2023年度150億円 → 2028年度300億円超見込み。
- **景気不透明下のコスト感応度。** 中小は予算がタイトで、ROIの明確さと低価格・明朗会計が購買の鍵。

### Social（社会）
- **会議のムダがコストとして可視化。** 1万人規模企業でムダな会議の損失は年間約15億円との試算。会議のムダ要因上位は「長時間」「議題不明瞭」「何も決まらない」。
- **議事録作成の負荷。** 「会議時間の2〜3倍かかっていた」との声があり、少人数の中小では1人当たりの事務負担が重い。
- **円滑な社内コミュニケーションが人材確保・生産性向上に寄与**することが中小企業白書でも示唆 → 「決定事項とタスクが明確に共有される」価値は採用・定着面でも訴求可。
- **AIへの心理的受容が向上。** 企業の78%がAIを業務活用、75%のマネージャーが部下のAI利用を推奨。

### Technological（技術）
- **音声認識・LLMの精度が実用域に。** 日本語の文字起こし・要約・タスク抽出（誰が・何を・いつまでに）が高精度で可能に。
- **API連携の成熟。** Slack / Teams / Chatwork / LINE WORKS 等への通知連携や、Zoom/Meet/Teamsからの自動録音取得が容易。
- **コモディティ化リスク。** 文字起こし・要約自体はOpenAI/Google等の汎用APIで誰でも実装可能 → 単なる「文字起こし＋要約」は差別化困難になりつつある。
- **MS Copilot等プラットフォーマーの脅威。** Teams標準の議事録機能が高度化し、標準機能に飲み込まれるリスク。

## 2. SWOT分析

| | プラス要因 | マイナス要因 |

### 戦略示唆（クロスSWOT）
- **強み×機会（攻め）：** 「タスク実行管理＋日本語＋中小特化」を、補助金活用（最大4/5補助）とセットで提案し、人手不足に悩む中小へ一気に展開。
- **弱み×脅威（守り）：** コモディティ化とプラットフォーマー脅威に対し、文字起こし精度では戦わず**「タスク実行完了率」という上位価値**で差別化。セキュリティ体制を早期に整え、規制強化を逆に信頼の武器にする。

## 3. 3C分析

### Customer（顧客・市場）
- **ターゲット：** 従業員50名以下の中小企業。情シス専任が少なく、1人が複数業務を兼務 → 事務負担削減・省人化ニーズが切実。
- **市場環境：** 正社員不足企業53.4%、人手不足倒産が過去最多。少人数で成果を出す必要性が高く、業務効率化SaaSへの需要が構造的に増加。
- **購買特性：** 予算がタイトで**ROIの明快さ・低価格・導入の容易さ**が決め手。情報システム部門が薄いため「設定不要・すぐ使える」が重要。**補助金（最大4/5補助）の活用可否**が導入の後押しになる。
- **痛点の本質：** 「議事録が作れない」より、**「会議で決まったタスクが担当者に伝わらず実行されない」**という会議後の実行不全。中小ほど属人化しており抜け漏れが致命的。

### Competitor（競合）

競合を「海外グローバル系」「国内特化系」「プラットフォーマー」の3群に分けて精査した。

#### ■ 競合別ディープダイブ

**1. Otter.ai（海外最大手）**
- 2025年11月にようやく日本語リアルタイム文字起こしへ対応し日本市場へ進出。ただし**議事録が読みづらく、アクションアイテムが抽象的で「非参加者が次に何をすべきか判断しにくい」**との実測評価。OtterPilot（営業向け自動化）は英語会議でのみ高精度。
- → 日本語のタスク粒度・実行支援は MeetMemo の狙いどころと正面から競合せず、**まだ穴が大きい**。

**2. Fireflies.ai（タスク機能が最強の海外勢）**
- **最大の脅威**。アクションアイテム自動抽出＋Asana/Slack/Salesforce等へ**自動タスク化・音声コマンドでタスク作成**まで実装。タスク実行連携では先行。
- 弱点は①日本語UI・国内サポートの薄さ、②**AIクレジット従量課金でコストが読めない**（中小の稟議に不向き）、③Slack/Asanaなど海外ツール前提でChatwork/LINE WORKS等は手薄。
- → MeetMemoは「Fireflies相当のタスク自動化を、日本語＋国内チャット＋明朗定額で」提供できれば直接的に上位互換となる。

**3. Notta（国内シェア上位の本命対抗）**
- 日本語精度・認知ともに国内トップ級（日経225の72%導入）、ISO27001:2022・SOC2 TypeII・AWS東京リージョンと**セキュリティも堅牢**＝中小の信頼面で強敵。
- 弱点はタスクが**「決定/要確認/タスク」のタグ付け止まり**で、担当者への自動アサイン・通知・実行追跡が弱い。連携もZapier依存の汎用。
- → 「文字起こし精度」で正面勝負すると不利。**タスク実行管理の深さ**でのみ差別化可能。

**4. Rimo Voice（純国産・低価格）**
- 日本語・対面商談に強く安価（1,500円〜）。ただし要約・AIエージェント中心で、**担当者へのタスク通知・実行追跡は主戦場でない**。価格競争相手にはなるが機能軸はずれている。

**5. Otolio（旧スマート書記／大手・自治体向け）**
- 高精度・決定事項/タスク抽出に対応し7,000社導入だが、**月額25,000円〜と高価格**でエンタープライズ・自治体寄り。**従業員50名以下の中小には価格が重く、ターゲットが異なる**＝MeetMemoと直接競合しにくい。

**6. MS 365 Copilot / Teams Premium（プラットフォーマー脅威）**
- Teamsを使う企業は標準機能でRecap＋Planner/To Doへのフォローアップ生成が可能になりつつある＝**長期的な最大の構造的脅威**。
- ただしCopilotは**30名で月900ドル規模**と高額かつライセンス前提で、Teams非利用・予算の薄い中小には即適用しにくい。Web会議横断（Zoom/Meet含む）や日本語タスク粒度では専用ツールに優位が残る。

#### ■ 競合分析からの結論
- **文字起こし・要約はコモディティ化**しており、Otter/Notta/Rimo/Otolio とも高精度で**差別化困難**。
- タスク自動化はFireflies/MS Copilotが先行するが、**いずれも「日本語の担当者アサイン精度 × 国内チャット通知 × 中小向け明朗価格」を同時に満たしていない**。
- 国内勢（Notta/Rimo）は日本語◎だが**タスク実行追跡が浅い**。Otolioは高価格で中小と非競合。
- → **空白地帯＝「日本語 × タスク実行管理（アサイン〜通知〜完了追跡）× 中小特化・明朗定額」**。ここがMeetMemoの唯一の勝ち筋。

### Company（自社）
- **提供価値：** 録音→議事録化→**タスク抽出→担当者への自動通知**まで一気通貫。「議事録を作る」のではなく「会議の決定を実行させる」ツール。
- **強み：** 中小特化で機能・UI・価格を最適化でき、意思決定が速い。日本語・国内チャットツール連携に集中投資可能。
- **弱み：** 知名度・実績不足、リソース制約、外部API依存による技術的参入障壁の低さ、録音データのセキュリティ／プライバシー対応が必須。
- **取るべきポジション：**
  1. **KPIを「タスク実行完了率」に置く**——文字起こし精度競争を回避し上位価値で勝負。
  2. **日本語×国内ツール連携（Slack/Teams/Chatwork/LINE WORKS）に最適化**——Otter/Firefliesの弱点を突く。
  3. **席数ベースの明朗定額＋補助金活用提案**——Fireflies従量課金の不透明さを回避し、IT導入支援事業者登録で中小への販路・信頼を獲得。

---

## まとめ（戦略の方向性）
PEST分析からは「人手不足×補助金後押し×AI受容性向上」という強い追い風と、「コモディティ化×プラットフォーマー×個人情報規制」という脅威が併存。SWOT・3Cを踏まえた勝ち筋は、

> **文字起こしでは戦わず、「会議の決定事項を担当者に届け、実行完了まで追跡する」実行管理プラットフォームとして、日本語・国内ツール連携・中小特化・明朗価格＋補助金活用で差別化する。**

セキュリティ／個人情報対応を早期に整備し、それを信頼の差別化要因へ転換することが持続的優位の条件となる。
